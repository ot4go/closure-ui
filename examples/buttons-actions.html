closure-ui — Buttons and actions

Demo of the button/action components: <closure-btn>, <closure-btn-item>, <btn-grid>, <target-closure>, <closure-template> and <status-buttons>.

This page is deliberately pure HTML — it has no stylesheet at all, so everything you see is the components' built-in default styling. For restyling via CSS variables see theming.html.

1. <closure-btn> — variants

The basic button supports an icon (icon), colour and size variants via class (primary, red, green, gray, small), a fixed width (width) and an icon-only mode (nolabel, with the tooltip taken from label). The host is display: block by default, so standalone buttons stack vertically; use <btn-grid> or <status-buttons> to arrange them.

Save Delete Approve Back Small Disabled Readonly

The last button has readonly: it is invisible but still occupies its space (visibility: hidden), useful to keep grid alignment. disabled, by contrast, is dimmed but visible. The gear button uses nolabel and shows its label as a tooltip on hover.

2. <closure-btn menu="…"> — dropdown menu

With the menu attribute the button becomes a dropdown whose panel hosts several <closure-btn-item>. Each item inherits the parent button's data-* attributes and can override them. Try it with the keyboard too: Enter/Space opens the panel, ↓/↑ move the focus, Enter activates and Esc closes.

Edit Export CSV Archive Delete (disabled) 
(activate a menu option to see its payload here)
<closure-btn id="menu-demo" menu="Row actions" icon="⋯" width="200" data-id="42">
  <closure-btn-item ct-role="edit"    icon="✎">Edit</closure-btn-item>
  <closure-btn-item ct-role="export"  icon="📥" data-format="csv">Export CSV</closure-btn-item>
  <closure-btn-item ct-role="archive" icon="🗄">Archive</closure-btn-item>
  <closure-btn-item ct-role="delete"  icon="🗑" disabled>Delete</closure-btn-item>
</closure-btn>

<script>
  // Items dispatch `btn-action` on the parent button, carrying the item in e.detail.source
  document.getElementById('menu-demo').addEventListener('btn-action', function (e) {
    var data = e.detail && e.detail.source ? e.detail.source.getBtnData() : null;
    console.log(data); // { ctRole, closureTemplate, sections: { '': { id: '42', ... } } }
  });
</script>

3. <btn-grid cols="…"> — button grid

Lays its children out in a CSS grid with cols columns (3 by default) and applies a "card" style to the inner <closure-btn> elements. With no-icon the icons are hidden and the buttons switch to compact text-only sizing.

4. Local action: client-action="set-value"

The button writes value into the .value property of the target (target-id, target-selector or target-selector-all) without a server round trip and without dispatching the btn-action event. Works fully offline.

5. <closure-btn free> — direct submit

With free the button bypasses the target-closure system: on click it builds a hidden form with all of its data-* attributes and submits it via POST to the given url (navigating to the response).

Sign out

Needs a server: the click performs a real POST and navigates to the URL, so it does not work from file://. In this demo the button is disabled on purpose; the real markup would be:

<!-- POSTs user=pablo to /logout and navigates to the response -->
<closure-btn free url="/logout" icon="🚪" class="red" data-user="pablo">
  Sign out
</closure-btn>

6. <target-closure> + <closure-template> + <form closure>

Complete wiring of a workflow: the closure gathers the forms by section (group-behavior="combine-sections"), the template declares the submit URL and how to package each section, and the button with ct-role="save" fires it. The closure also tracks the dirty state: type into any field and the dot appears (dirty-show="show").

⚠ No server available: the POST to /api/users/save failed (expected in this offline demo).

Profile ●

Save

What works offline: the change tracking (dirty dot) and the form packaging. The actual POST needs an endpoint at /api/users/save; without a server, pressing "Save" shows the error message declared in <template-response-fail type="no-response">.

<target-closure name="edit-user" group-behavior="combine-sections">
  <closure-template name="save">
    <template-url url="/api/users/save" method="POST"
                  send-behavior="fetch" ct-role="save"
                  response-target-id="save-result"></template-url>
    <template-section section="profile" mode="prefix"></template-section>
    <template-section section="prefs"   mode="prefix"></template-section>
    <template-field name="origin" value="demo"></template-field>
    <template-dirty-clean result="ok" templates="save"></template-dirty-clean>
    <template-response-fail type="no-response">
      <p>Connection error</p>
    </template-response-fail>
  </closure-template>

  <h3>Profile <span dirty-show="show">●</span></h3>

  <form closure section="profile">
    <input type="text" name="name">
    <input type="text" name="email">
  </form>

  <form closure section="prefs">
    <input type="text" name="language">
  </form>

  <closure-btn ct-role="save" icon="💾" class="primary">Save</closure-btn>

  <div id="save-result"></div>
</target-closure>

7. <status-buttons> — auto-laying group

Button group designed to live inside a <closure-status-bar> (a flex container that stretches it to full height and re-themes the inner buttons). It automatically picks the column count that minimises empty cells; a child with stretch-priority="0" is the preferred candidate to stretch and fill the trailing gap. In a wide bar all buttons fit in one row:

Approve Reject Hold Review Defer

The same bar with the group constrained through its own wr="*,180px" width-range attribute (no page CSS involved): the group switches to grid mode and the stretch-priority="0" button absorbs the empty cell:

Approve Reject Hold Review Defer 
<closure-status-bar type="white">
  <label>Order #1042</label>
  <status-buttons>            <!-- add wr="*,180px" to cap its width -->
    <closure-btn ct-role="approve">Approve</closure-btn>
    <closure-btn ct-role="reject">Reject</closure-btn>
    <closure-btn ct-role="hold">Hold</closure-btn>
    <closure-btn ct-role="review">Review</closure-btn>
    <!-- preferred candidate to absorb the trailing gap -->
    <closure-btn ct-role="defer" stretch-priority="0">Defer</closure-btn>
  </status-buttons>
</closure-status-bar>
