closure-ui — Data components

Self-contained demos (inline static data, no server) of <closure-data-grid>, <closure-filter-bar>, <closure-row-viewer>, <closure-data-source> and <data-map>.

This page is deliberately pure HTML — it has no stylesheet at all, so everything you see is the components' built-in default styling. For restyling via CSS variables see theming.html.

1. Basic grid with inline data

Rows are declared with <g-row>/<g-col> inside the grid itself. Columns are defined with <grid-col> (name, label, width) and <grid-key> declares each row's identity. With page-size="5" and 12 rows, the pagination controls show up in the footer.

⌨ Keyboard navigation: click the grid to focus it; the ↑/↓ arrow keys move the focused row (row-focus event), Enter promotes it to selected (row-select event), and PageUp/PageDown change page. This grid uses the default continuous scroll disposition, so the mouse wheel scrolls the page normally; see scroll modes below for the window disposition where the wheel flips pages instead.

1b. Scroll disposition & <grid-layout>

A grid is either a continuous flow (its natural height; the mouse wheel scrolls the document so you can reach whatever is below it) or a height-bounded window (the wheel over the body flips pages). Set it explicitly with scroll="window|continuous", or let it be inferred: page-size="auto" or a max-rows cap ⇒ window, otherwise continuous.

Window — hover the body and spin the wheel: it pages instead of scrolling the document.

3. <closure-filter-bar> paired with a grid

The bar dispatches filter-change on the element named by target. On a grid with static data the default filter mode is local: select fields are compared by exact equality against the row, and the special text field q performs a substring search across all fields. Active values show as removable chips, and <filter-preset> entries apply value sets with one click.

4. Master–detail with <g-detail> and detail-rows

Each row of the master grid embeds a <g-detail name="turnos"> block holding its child rows; that block becomes an array field of the row and does not count as a top-level row (the master's footer counter shows 4 records, not 4 + shifts). The detail grid declares detail-of="gridDias" and detail-rows="turnos": when a master row is selected (row-select, the default detail-event trigger), the detail fills in from the embedded array with no request at all.

5. <closure-row-viewer> bound to a grid

The viewer listens to row-select and row-focus on the grid named by target and projects the row onto its descendants: bind="field" writes the value, bind-show/bind-hide show or hide based on conditions (field or field=value) and bind-crlf="<br>" turns line breaks in the value into HTML. Since cells arrive as text, the value "0" (and "false") counts as falsy: select “Marta Gil” (active = 0) to see it.

6. <closure-data-source> with cascading selects

An invisible container declares the rows (<g-row>/<g-col>) and each <observed-select> describes how to populate a <select>: key-field/label-field feed the options (deduplicated by key), filter-control/filter-field build the country → province cascade, selected-value sets the initial selection and blank-value-key prepends a blank option. Change the country and watch the province list repopulate.
