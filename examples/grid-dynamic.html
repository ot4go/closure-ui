closure-ui — Dynamic grid with a backend

This page is not self-contained: the grids fetch their rows from a small Go server. Run it and open this page through it:

cd examples/server
go run .
# then open http://localhost:8099/grid-dynamic.html

Opening this file directly (file://) will show empty grids and a fetch error — there is no endpoint to call. Every other example page works offline; this one is the exception that demonstrates dynamic mode.

The rows come from examples/server/data/people.json — edit that file (no recompile) to change what the demo serves.

Like the rest, this page is pure HTML — no stylesheet — so what you see is the components' built-in default styling.

1. Server-side pagination

In dynamic mode the grid fetches one page at a time. The <query-definition> declares the endpoint and the <query-param> children map the grid's paging state onto the request: bind="grid.offset" and bind="grid.limit". The server replies with { data:[…], result:{ total, offset, eof } } and the grid renders that page, deriving the pager from total.

2. Filtering wired to the backend

A paired <closure-filter-bar> targets the grid. Each <filter-field> value reaches the server through a matching <query-param bind="filter.X">; changing a filter re-fetches page 1. This bar also shows the two filter-field attributes added recently: placeholder (per-field hint) and default (seed value on first open) — the Name field opens pre-filled with Ana.
