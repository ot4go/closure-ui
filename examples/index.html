closure-ui — examples

Self-contained demo pages that load the compiled bundle (examples/closure-ui.js, refreshed by each build). They work when opened directly in a browser, no server needed; the few demos that require an endpoint say so in their section.

The demo pages are pure HTML — no stylesheets — so what you see is the components' built-in default styling. The only page with CSS is the theming demo, whose subject is CSS.

  • Buttons & actions
    <closure-btn> (variants, dropdown menu, client-action, free), <closure-btn-item>, <btn-grid>, <status-buttons> and the full <target-closure> + <closure-template> wiring with dirty tracking.

  • Grid & data
    <closure-data-grid> (pagination, scroll modes & <grid-layout>, mapped columns, master–detail), <closure-filter-bar>, <closure-row-viewer>, <closure-data-source> and <data-map>.

  • Dynamic grid (backend) — needs the demo server
    <closure-data-grid> in dynamic mode: server-side pagination and backend-wired filtering via <query-definition> / <query-param>. Run go run . in examples/server and open it through http://localhost:8099/.

  • Forms
    <closure-form-row>/<closure-form-field>, <credential-pwd> (including enter-btn-id), <closure-checkbox-tree>/<closure-checkbox-group> with <closure-summary>, and <fingerprint-hands>.

  • Status, tabs & lightbox
    <closure-status-bar> with its parts (<status-msg>, <status-kv>, <status-part>, <status-buttons>), <closure-tab-bar>, <closure-lightbox> (MsgAlert/MsgConfirm), <clock-display>, <session-keep-alive>, <closure-summary> and <signal-event>.

  • Lazy iframe panel
    <closure-lazy-iframe>: collapsible panel whose iframe src is only assigned on first expand — placeholder children, toggle()/unload() and the lzi-* events.

  • Theming with CSS variables
    The one page that does use CSS: restyling components through their public CSS variables (shared tokens, --form-btn-*, tab palette).

Full per-component documentation lives in doc/closure-ui.md.
