closure-ui — status, tab and dialog components

Self-contained demo (works from file://) of <closure-status-bar>, <closure-tab-bar>, <closure-lightbox>, <clock-display>, <session-keep-alive>, <closure-summary> and <signal-event>.

This page is deliberately pure HTML — it has no stylesheet at all, so everything you see is the components' built-in default styling. For restyling via CSS variables see theming.html.

1. <closure-status-bar>

Horizontal status bar that lays out its children in a flex layout and acts as a colour preset host: the type attribute paints background and border, and re-themes the inner buttons. It is composed from <label>, <status-msg>, <status-kv>, <status-part> and <status-buttons>.

2. <closure-tab-bar> + <closure-tab>

Tab control without Shadow DOM. Each <closure-tab> provides name, label, icon and optionally disabled, toggle (a checkbox on the button that enables/disables the panel and writes 0/1 into the hidden input named by toggle-target) and show-source (an external checkbox that shows/hides the tab). Switching tabs fires the tab-change event with { name, prev }. The active panel ships with a default frame (padding, border, background) so the tabs look connected with no page CSS.

3. <closure-lightbox>

Modal based on the native <dialog>. It can be opened programmatically with open({ title, content, buttons }); on close it emits lb-close with the action of the button that was pressed. The static helpers ClosureLightbox.MsgAlert(msg, title?) and ClosureLightbox.MsgConfirm(msg, title?) (the latter returns a Promise<boolean>) spawn throw-away lightboxes.

(the result will appear here)
<closure-lightbox id="demo-lb" title="Initial title"></closure-lightbox>

<script>
  // Programmatic open with title, content and footer buttons
  const lb = document.getElementById('demo-lb');
  lb.open({
    title: 'Confirm submission',
    content: '<p>Send the report?</p>',
    buttons: [
      { label: 'Cancel', action: 'cancel' },
      { label: 'Send',   action: 'send', primary: true },
    ],
  });
  lb.addEventListener('lb-close', e => console.log(e.detail.action));

  // Static helpers
  ClosureLightbox.MsgAlert('Saved successfully', 'Notice');
  ClosureLightbox.MsgConfirm('Delete user 42?').then(ok => {
    if (ok) doDelete();
  });
</script>

4. <clock-display>

Live wall clock. It normally syncs with the server via GET /api/time; since this demo has no server, it uses the browser's LOCAL clock — the documented fallback when synchronisation fails (here we additionally set window.closure_clock_skip_sync_time = 1 to skip the network call entirely, as the doc recommends for mockups). That is also why the «Server Time» / ● label does not appear: it is only painted after a successful sync.

5. <session-keep-alive>

Idle countdown. Without extend-url, clicking the button only resets the countdown on the client (and emits session-extended). When it reaches zero it emits session-expired; since there is no expire-url either, no navigation happens. When the remaining time drops below warn-at the button turns red (the warn attribute). The activity-reset variant resets on any mouse, keyboard or touch activity — so in practice it almost never expires while you are on the page.

timeout=90, warn-at=60 (click to extend):

+ activity-reset (resets on activity):

(session-extended / session-expired events will appear here)
<session-keep-alive timeout="90" warn-at="60"></session-keep-alive>
<session-keep-alive timeout="90" warn-at="60" activity-reset></session-keep-alive>

<script>
  ska.addEventListener('session-extended', () => log('extended'));
  ska.addEventListener('session-expired',  () => log('expired'));
</script>

<!-- In production, with a server: -->
<session-keep-alive timeout="900" warn-at="120"
  extend-url="/admin/keep-alive" expire-url="/admin/logoff"
  data-sid="abc123" activity-reset></session-keep-alive>

6. <closure-summary>

Read-only renderer. It is normally paired with a «source» element (source="id" attribute) that implements getSummaryHTML(); here we use it standalone by calling its refresh(html) method from a button.

<closure-summary id="demo-summary"></closure-summary>

<script>
  document.getElementById('demo-summary').refresh(
    '<ul><li><strong>Users:</strong> 12</li>' +
    '<li><strong>Active:</strong> 9</li></ul>'
  );
</script>

7. <signal-event>

One-shot event dispatcher for declarative «server-push»: when parsed (in connectedCallback) it reads its name and its data-* attributes, dispatches a CustomEvent on document (each data-* becomes a key of event.detail, with the data- prefix stripped) and removes itself from the DOM. The listener must be registered before the element is parsed. Here we simulate it by injecting the element via JS, just like a streamed server HTML response would.

(the event detail will appear here)
<script>
  // 1) Subscribe BEFORE the signal arrives
  document.addEventListener('demo-notice', (e) => {
    console.log(e.detail['token'], e.detail['employee-id']);
  });
</script>

<!-- 2) The server emits this in its HTML response;
        it fires when parsed and removes itself -->
<signal-event name="demo-notice"
              data-token="abc123"
              data-employee-id="42"></signal-event>
